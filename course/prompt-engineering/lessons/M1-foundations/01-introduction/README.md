## What is Prompt Engineering?

Prompt engineering is the art and science of designing inputs (prompts) to get optimal outputs from AI language models.

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│     PROMPT      │ ──► │   AI MODEL      │ ──► │    OUTPUT       │
│  (Your Input)   │     │  (Processing)   │     │  (Response)     │
└─────────────────┘     └─────────────────┘     └─────────────────┘
```

### Why It Matters

| Aspect | Bad Prompt | Good Prompt |
|--------|------------|-------------|
| Clarity | "fix this" | "Fix the bug in this Python function that causes IndexError when the list is empty" |
| Specificity | "write code" | "Write a Python function that calculates the median of a list of numbers, handling edge cases for empty lists and even/odd lengths" |
| Context | "analyze" | "Analyze this customer feedback for sentiment, extracting key themes and suggesting 3 actionable improvements" |

## The Prompt Engineering Mindset

### Core Principles

1. **Be Specific**: Vague prompts → vague outputs
2. **Provide Context**: Help the model understand your intent
3. **Iterate**: First attempt rarely perfect
4. **Test Systematically**: Measure, don't guess

### The PIERS Framework

```
P - Purpose: What do you want?
I - Information: What context is needed?
E - Examples: Show, don't just tell
R - Role: Who should the AI be?
S - Style: How should output look?
```

## Your First Prompts

### Example 1: Basic Prompt

**❌ Poor**:
```
Write code to sort
```

**✅ Better**:
```
Write a Python function to sort a list of integers in ascending order
using the quicksort algorithm. Include comments explaining each step.
```

### Example 2: With Examples (Few-Shot)

```
Convert these sentences to formal business English:

Informal: "The meeting's at 3, don't be late."
Formal: "The meeting is scheduled for 3:00 PM. Please ensure punctual attendance."

Informal: "We need to talk about the budget cuts."
Formal: ?
```

## Key Terminology

| Term | Definition |
|------|------------|
| **Prompt** | Input text given to an AI model |
| **Completion** | Output generated by the model |
| **Context Window** | Maximum text the model can process |
| **Token** | Basic unit of text (~4 chars in English) |
| **Temperature** | Controls randomness (0 = deterministic, 1 = creative) |
| **System Prompt** | Instructions defining AI behavior |
